# 网站后台维护操作手册

## 1. 进入后台

本地完整后端地址：

```text
http://127.0.0.1:5000/admin
```

如果看到登录页，输入 `.env` 里的 `ADMIN_PASSWORD`。本地默认是 `admin123`，正式部署前必须修改。

注意：`http://127.0.0.1:3456/` 是静态预览，只能看页面，不能维护模型，也不能读取后台数据库。

## 2. 常用菜单怎么用

### 分类管理

用于控制前台左侧/模型分组，例如：

- 图像生成
- 图像编辑
- 视频生成
- 音频/语音
- 分析与提示词
- 任务工具

可以拖拽排序。

### 模型管理

用于新增或修改一个模型入口。重点字段：

- 模型 ID：网站内部稳定 ID，后续不要随便改。
- 分类：模型属于哪个分组。
- 名称：用户看到的模型名。
- 供应商：例如 Google / Tikpan、OpenAI / Tikpan。
- API 类型：决定后端怎么调用。
- 端点：上游接口路径。

### 字段管理

用于配置用户前台能填什么参数。原则：

- 能做下拉框就不要让用户手填。
- 常用字段放前面，高级字段放后面。
- 用户看不懂的字段要改成中文业务名。
- 图片/参考图要设置最大数量。

### 供应商渠道

用于配置多个上游中转站或供应商：

- `tikpan-default`：你的默认 Tikpan 渠道。
- `base_url`：供应商域名。
- `api_key`：供应商 key。
- `priority`：优先级，数字越小越优先。
- `timeout_seconds`：超时时间。

后续你接多家中转站，就在这里新增渠道。

### 计价规则

用于控制用户扣费：

- 按分辨率：适合图片模型。
- 固定每次：适合视频、音乐。
- 按字段数量：适合 TTS 字符数、token 预算、秒数。

上线前要保证售价覆盖上游成本。

### 余额流水

用于查账：

- 充值入账
- 生成扣费
- 失败退款
- 代理申请费用
- 人工调整

用户问“为什么扣费”时，优先看这里。

## 3. 同步 ComfyUI 节点到网站

如果你在 ComfyUI 里新增或修改 Tikpan 节点，推荐先更新：

```text
web_app/catalog/tikpan_nodes.py
```

然后运行：

```powershell
D:\ComfyUI-aki-v2\python\python.exe -B web_app\scripts\sync_tikpan_nodes.py
```

同步后再进后台检查模型、字段、价格、路由。

## 4. 新模型更新流程

1. 先在 ComfyUI 节点里跑通。
2. 把参数整理成中文字段和下拉选项。
3. 写入 `web_app/catalog/tikpan_nodes.py`。
4. 运行同步脚本。
5. 后台检查字段顺序和默认值。
6. 配置供应商渠道和计价规则。
7. 用小额测试任务验证扣费、失败退款、结果返回。
8. 再开放给用户。

## 5. 本地开发常用命令

启动完整后端：

```powershell
cd D:\ComfyUI-aki-v2\ComfyUI\custom_nodes\ComfyUI-Tikpan-Pro\web_app
.venv\Scripts\python.exe app.py
```

同步模型：

```powershell
cd D:\ComfyUI-aki-v2\ComfyUI\custom_nodes\ComfyUI-Tikpan-Pro
web_app\.venv\Scripts\python.exe web_app\scripts\sync_tikpan_nodes.py
```

部署检查：

```powershell
cd D:\ComfyUI-aki-v2\ComfyUI\custom_nodes\ComfyUI-Tikpan-Pro
web_app\.venv\Scripts\python.exe web_app\scripts\deploy_check.py
```

## 6. 正式上线前必须做

- 修改 `ADMIN_PASSWORD`。
- 修改 `FLASK_SECRET` 和 `TIKPAN_SECRET`。
- 配好 `TIKPAN_API_KEY`。
- 配好 OSS/CDN，避免大图视频压垮服务器带宽。
- 确认每个主推模型都有价格和渠道。
- 做一次失败退款测试。
