# Tikpan Web 部署与迁移指南

## 1. 机房选择

面向中国用户，同时要调用 OpenAI、Google Gemini、Anthropic、字节、MiniMax 等上游时，推荐：

1. 首选新加坡：上游 API 风控更友好，国际带宽更充足，适合作为主业务服务器。
2. 香港适合作为接入层：国内访问延迟低，但上游大模型 API 的区域/IP 风险更高，不建议把所有上游请求都压在香港主机上。
3. 单机起步可以用新加坡 4 核 4G + 30M CN2/优化线路。文本和 API 网关足够，图片/视频结果必须尽快走 OSS/CDN。

推荐链路：

```text
国内用户 -> CDN/WAF/域名 -> Nginx -> Tikpan Web -> tikpan.com/上游 API
                                      -> OSS/CDN 返回图片/音频/视频 URL
```

## 2. 4H4G 30M 是否够用

够起步，但要按下面原则使用：

- 文本、计费、鉴权、路由、日志：4H4G 足够。
- 30M 带宽不要直接下发大图、大音频、大视频。
- 生成结果先上传 OSS，再返回 CDN URL。
- 上传参考图要限制数量和大小，前端/节点侧尽量压缩。
- SQLite 适合早期验证；有稳定收入后迁移 PostgreSQL/MySQL。

## 3. 环境变量

复制配置：

```bash
cd web_app
cp .env.example .env
```

必须修改：

```bash
APP_ENV=production
PUBLIC_BASE_URL=https://你的域名
TIKPAN_API_KEY=sk-你的key
TIKPAN_SECRET=随机长字符串
FLASK_SECRET=随机长字符串
ADMIN_PASSWORD=强密码
TRUST_PROXY=true
```

开启 OSS/CDN：

```bash
OSS_ENABLED=true
OSS_BUCKET=你的bucket
OSS_ENDPOINT=oss-ap-southeast-1.aliyuncs.com
OSS_KEY_ID=你的AccessKeyId
OSS_KEY_SECRET=你的AccessKeySecret
OSS_PREFIX=tikpan-web
OSS_CDN_DOMAIN=https://cdn.你的域名
```

服务器和 OSS 尽量选同云厂商同区域，例如新加坡 ECS + 新加坡 OSS，这样上传对象存储更稳定。

## 4. Docker 部署

Flask 官方文档强调，生产环境不要直接使用内置开发服务器，应该放在 WSGI 服务器和反向代理之后；这里采用 Gunicorn + Nginx。Docker Compose 的 `.env` 适合启动期管理环境变量，更高安全要求时再升级为云厂商密钥管理或 Docker secrets。Nginx 反代使用 `proxy_pass` 转发到本机 Gunicorn。

安装 Docker 后：

```bash
cd web_app
docker compose up -d --build
docker compose logs -f
```

服务默认监听本机：

```text
127.0.0.1:5000
```

用 Nginx 对外反代。

## 5. Nginx

复制示例：

```bash
sudo cp web_app/deploy/nginx-tikpan.conf /etc/nginx/conf.d/tikpan.conf
sudo nano /etc/nginx/conf.d/tikpan.conf
```

把 `your-domain.com` 改成你的域名，然后：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

HTTPS 建议用 Certbot 或云厂商证书。生产环境必须启用 HTTPS。

如果前面套 Cloudflare、宝塔、SLB 或其他 CDN/WAF，`.env` 里保持 `TRUST_PROXY=true`，让 Flask 正确识别真实协议和域名。

## 6. 数据迁移

项目提供了可移植数据库脚本，避免换电脑/换服务器时丢后台配置。

导出配置：

```bash
cd web_app
python scripts/portable_db.py export --db data/tikpan.db --out data/config.export.json
```

导入到新服务器：

```bash
cd web_app
python scripts/portable_db.py import --db data/tikpan.db --in data/config.export.json
```

注意：

- `data/tikpan.db`、`.env`、OSS Key、API Key 不应提交 Git。
- `config.export.json` 适合迁移模型、字段、价格、渠道配置；敏感字段按脚本规则处理。
- 正式上线后建议每天备份 `web_app/data/tikpan.db` 到 OSS 私有目录。

## 7. 上线检查清单

先运行：

```bash
python scripts/deploy_check.py
```

再检查：

- 域名解析到服务器。
- Nginx HTTPS 可访问。
- `.env` 密钥都已改成强随机值。
- `OSS_ENABLED=true` 并确认返回的是 CDN URL。
- 后台 `/admin` 密码已修改。
- 充值回调和 OAuth 回调域名使用 `PUBLIC_BASE_URL`。
- 生成一张小图，检查余额扣费、generation_logs、OSS 文件、CDN URL。
- 关闭 debug，不暴露 Flask 开发服务器。

## 8. 后期维护

每周：

- 查看错误日志和退款日志。
- 检查 OSS/CDN 流量成本。
- 检查供应商渠道成功率和平均耗时。
- 导出一次后台配置。

每月：

- 轮换上游 API Key。
- 复核模型价格和渠道成本。
- 清理过期输出文件。
- 做一次恢复演练：用备份数据库在新机器启动。

## 9. 后续升级方向

第一阶段：

- 单台新加坡服务器 + SQLite + OSS/CDN。

第二阶段：

- Redis 队列处理长任务。
- PostgreSQL/MySQL 替代 SQLite。
- 后台增加渠道健康检查、失败自动切换。

第三阶段：

- 香港/国内优化接入层 + 新加坡核心 API 层。
- 多区域备份和灰度发布。
- 独立计费流水、发票、代理分润结算。
